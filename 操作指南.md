# 操作指南

## 一、平台介绍

语音Badcase流转平台是一个专门用于管理和跟踪语音识别问题的工具。通过这个平台，您可以：

- 📝 记录语音识别中发现的问题
- 🎵 关联问题音频文件
- 🏷️ 设置问题优先级
- 📊 跟踪问题处理状态
- 🔍 快速搜索和过滤问题
- 📈 统计和分析问题数据

## 二、界面说明

### 顶部功能区

```
┌─────────────────────────────────────────────────────────┐
│ 🔍 搜索框 | 状态筛选 | 优先级筛选 | 日期范围 | ➕ 新建问题  │
└─────────────────────────────────────────────────────────┘
```

- **搜索框**：输入关键词搜索问题文本
- **状态筛选**：按问题处理状态过滤
- **优先级筛选**：按优先级等级过滤
- **日期范围**：按反馈日期筛选
- **新建问题**：创建新的badcase记录

| 列名 | 说明 | 操作 |
|------|------|------|

## 三、基本操作

### 3.1 新建问题

**步骤**：

1. 点击右上角 **"新建问题"** 按钮
2. 填写表单信息：

   **必填项**：
   - 问题文本：输入完整的问题文本内容
   - 问题描述：简要描述问题
   - 修复优先级：选择 P00/P0/P1/P2
   - 问题状态：选择当前状态

   **选填项**：
   - 音频URL：如果有音频文件，输入URL
   - 问题体描述：更详细的问题说明
   - 反馈来源：如"测试"、"用户反馈"等
   - 反馈日期：选择日期
   - 反馈人：输入姓名
   - 创建人：输入姓名

3. 点击 **"确定"** 保存

**示例**：

```
问题文本：farmer 后边的/mer/没有发出/m/r的音
问题描述：谜错
修复优先级：P00
问题状态：修复中
反馈日期：2025-08-22
反馈人：张未
创建人：张未
```

### 3.2 编辑问题

**步骤**：

1. 找到要编辑的记录
2. 点击操作列的 **"编辑"** 按钮
3. 修改需要更新的字段
4. 点击 **"确定"** 保存

**常见编辑场景**：

- 更新问题状态（如从"修复中"改为"已上线并验证"）
- 调整优先级
- 补充详细描述
- 添加音频URL

### 3.3 搜索问题

**方法1：文本搜索**

在顶部搜索框输入关键词，系统会实时搜索问题文本字段。

示例：
- 输入 "farmer" → 查找所有包含farmer的问题
- 输入 "发音" → 查找所有包含发音的问题

**方法2：状态筛选**

点击"问题状态"下拉框，选择要查看的状态：

- 修复中：正在处理的问题
- 待确认：需要确认的问题
- 已上线并验证：已修复并验证的问题
- 已关闭：已关闭的问题
- 停顿：暂停处理的问题

**方法3：优先级筛选**

点击"修复优先级"下拉框，选择优先级：

- P00：最高优先级，需立即处理
- P0：高优先级
- P1：中优先级
- P2：低优先级

**方法4：日期范围**

点击"日期范围"选择器，选择起止日期。

**组合筛选**：

可以同时使用多个筛选条件，系统会取交集显示结果。

示例：搜索"2025年8月"的"P00"级别"修复中"的问题。

### 3.4 播放音频

**步骤**：

1. 找到有音频的记录（音频列显示播放按钮）
2. 点击 🔵 播放按钮
3. 再次点击暂停播放

**注意事项**：

- 同一时间只能播放一个音频
- 音频文件需要浏览器支持的格式（mp3/wav/ogg）
- 音频URL必须可访问

### 3.5 排序

点击表头列名可以进行排序：

- 点击"反馈日期"：按日期排序
- 点击其他列：触发相应排序

### 3.6 分页

- 表格底部显示分页控件
- 默认每页显示20条记录
- 可以切换页码查看更多数据
- 显示总记录数

## 四、优先级说明

### P00 - 最高优先级 🔴

**特征**：
- 严重影响用户体验
- 高频出现
- 需要立即修复

**示例**：
- 常用单词发音错误
- 音频完全无法播放
- 系统崩溃

### P0 - 高优先级 🟠

**特征**：
- 明显的错误
- 影响部分用户
- 需要尽快修复

**示例**：
- 特定场景下的发音问题
- 功能异常但有变通方法

### P1 - 中优先级 🟡

**特征**：
- 较小的问题
- 影响范围有限
- 可以计划修复

**示例**：
- 偶发的音质问题
- 低频词汇发音偏差

### P2 - 低优先级 🟢

**特征**：
- 轻微问题
- 影响很小
- 可以延后处理

**示例**：
- 优化建议
- 极低频词汇问题

## 五、状态说明

### 修复中 🔧

问题已确认，正在修复中。

### 待确认 ⏳

问题已记录，等待确认是否需要修复。

### 已上线并验证 ✅

问题已修复，已上线并通过验证。

### 已关闭 ⛔

问题已关闭（可能是重复、无效或不修复）。

### 停顿 ⏸️

问题修复暂时停止，可能因为优先级调整或其他原因。

## 六、工作流程示例

### 场景1：发现新问题

1. 测试人员发现问题
2. 点击"新建问题"创建记录
3. 填写问题信息，设置优先级P00，状态"待确认"
4. 开发人员查看并确认
5. 更新状态为"修复中"
6. 修复完成后更新为"已上线并验证"

### 场景2：批量查看问题

1. 使用状态筛选查看所有"修复中"的问题
2. 按优先级排序，优先处理P00问题
3. 逐个更新处理进度

### 场景3：问题追溯

1. 在搜索框输入关键词
2. 找到历史记录
3. 查看音频和描述
4. 分析问题原因

## 七、最佳实践

### 7.1 创建记录

✅ **好的做法**：
- 问题文本尽量完整，包含上下文
- 描述准确、简洁
- 优先级评估合理
- 及时上传音频文件

❌ **不好的做法**：
- 问题描述过于简单："有问题"
- 优先级随意设置
- 缺少关键信息

### 7.2 更新状态

✅ **好的做法**：
- 状态变更及时更新
- 按实际进度修改
- 完成时及时标记

❌ **不好的做法**：
- 长期不更新状态
- 状态与实际不符

### 7.3 搜索技巧

✅ **好的做法**：
- 使用关键词搜索
- 组合多个条件
- 利用日期范围

❌ **不好的做法**：
- 手动翻页查找
- 不使用筛选功能

## 八、常见问题

### Q1：如何上传音频文件？

A：目前支持两种方式：
1. 直接输入音频文件URL
2. 通过API上传音频（开发中）

### Q2：能否批量操作？

A：当前版本暂不支持，后续版本会添加。

### Q3：数据能否导出？

A：当前版本暂不支持，后续版本会添加Excel/CSV导出功能。

### Q4：如何删除记录？

A：出于数据安全考虑，当前版本暂不提供删除功能，如需删除请联系管理员。

### Q5：支持移动端吗？

A：界面采用响应式设计，可以在移动设备上使用，但建议在电脑上操作以获得最佳体验。

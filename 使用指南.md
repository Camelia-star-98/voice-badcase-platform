## ✨ 新功能说明

### 1️⃣ 出现位置功能 🆕🆕

在新建Badcase时，现在需要选择问题的**出现位置**，系统会根据选择显示不同的ID输入框：

#### 出现位置选项：
- **全程TTS做课部分** - 问题出现在全程TTS做课场景
  - 需要填写：**全程TTS课节ID**
  - 🆕 **不需要填写：问题模型ID**（此场景下不需要）
- **行课互动部分** - 问题出现在行课互动场景
  - 需要填写：**CMS课节ID**
  - 需要填写：**问题模型ID**

#### 表格显示：
- **出现位置** 列 - 显示问题出现的场景
- **课节ID** 列 - 根据场景显示对应的ID

### 2️⃣ ID 排序一致性 🆕

**Badcase列表** 和 **流转状态** 两个页面现在都按照 **ID 从新到旧（降序）** 自动排列：

- ✅ 最新添加的记录（ID最大）始终显示在最前面
- ✅ 两个页面的记录顺序完全一致
- ✅ 搜索、筛选后仍保持ID降序
- ✅ 例如：BC0010、BC0007、BC0005、BC0003、BC0001

### 3️⃣ 数据同步

**Badcase列表** 和 **流转状态** 两个页面的数据现在是完全同步的：

- ✅ 在 Badcase 列表新增数据 → 流转状态页面立即显示
- ✅ 在流转状态修改状态 → Badcase 列表立即更新
- ✅ 在流转状态删除数据 → Badcase 列表同步删除

### 4️⃣ 删除功能

在 **流转状态** 页面，每条记录的操作列现在有：

- 👁️ **查看** - 查看详细信息
- ▶️ **播放** - 播放音频（如果有）
- 🗑️ **删除** - 删除该条记录（需要确认）

### 5️⃣ 数据持久化

所有操作都会自动保存到浏览器的本地存储：

- 新增、修改、删除操作都会被保存
- 刷新页面后数据不会丢失
- 数据保存在 `localStorage` 中

## 📖 使用流程

### 场景1：新增 Badcase

1. 进入 **Badcase列表** 页面
2. 点击右上角 **"新建Badcase"** 按钮
3. 填写表单：
   - 选择学科
   - **🆕 选择出现位置**（全程TTS做课部分 / 行课互动部分）
   - **🆕 根据出现位置填写对应的ID**
     - 全程TTS做课部分 → 填写"全程TTS课节ID"
     - 行课互动部分 → 填写"CMS课节ID"
   - 填写问题提报人
   - **🆕 如果选择"行课互动部分"** → 选择问题模型ID
   - **🆕 如果选择"全程TTS做课部分"** → 不需要填写问题模型ID
   - 选择分类
   - 填写期望修复时间
   - 填写问题描述
   - （可选）上传音频文件
4. 点击 **"提交"**
5. ✅ 新数据立即显示在列表中
6. 切换到 **流转状态** 页面，新数据也会显示

### 场景2：修改状态

1. 进入 **流转状态** 页面
2. 输入密码 `1222` 进入页面
3. 在 **"流转状态"** 列的下拉菜单中选择新状态：
   - 🔄 待处理
   - ⚙️ 算法处理中
   - 🔧 工程处理中
   - ✅ 已解决
4. ✅ 状态立即更新
5. 切换到 **Badcase列表** 页面，状态已同步

### 场景3：删除记录

1. 进入 **流转状态** 页面
2. 输入密码 `1222` 进入页面
3. 找到要删除的记录
4. 点击 **"删除"** 按钮
5. 在弹出的确认对话框中点击 **"确认"**
6. ✅ 记录被删除
7. 切换到 **Badcase列表** 页面，记录也被删除

## ⚠️ 注意事项

1. **数据存储**
   - 数据存储在浏览器本地
   - 不同浏览器的数据不共享
   - 清除浏览器缓存会丢失数据

2. **删除操作**
   - 删除操作不可恢复
   - 请谨慎操作

3. **密码验证**
   - 流转状态页面需要密码验证
   - 密码在当前会话中有效
   - 关闭浏览器后需要重新输入

4. **状态说明**
   - **待处理** - 新提交的问题
   - **算法处理中** - 算法团队正在处理
   - **工程处理中** - 工程团队正在处理
   - **已解决** - 问题已经解决

## 🐛 常见问题

**Q: 刷新页面后数据消失了？**
A: 检查浏览器是否禁用了 localStorage，或者是否清除了缓存。

**Q: 两个页面的数据不同步？**
A: 尝试刷新页面。如果仍有问题，清除 localStorage 重新开始。

**Q: 忘记密码了？**
A: 流转状态页面的密码是 `1222`。

**Q: 如何导出数据？**
A: 当前版本暂不支持导出，可以在浏览器控制台执行：
```javascript
console.log(JSON.stringify(JSON.parse(localStorage.getItem('badcaseList')), null, 2));
```
